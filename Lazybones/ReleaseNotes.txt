v.1.1.1
   - Fix: auto-update restarts no longer drop the in-flight cycle. Velopack's default ApplyUpdatesAndRestart bypassed Avalonia's window-close events, so MainWindowViewModel.Dispose never ran and the cycle's state was silently lost (visible as missing standing cycles in your history after an update). The app now uses WaitExitThenApplyUpdates + a graceful Avalonia shutdown so cycle state survives the patch.
   - Pauses are now recorded as part of each cycle's history: screen-lock auto-pauses, manual pauses, and app-shutdown spans (graceful or not). Cycle stats still exclude pause time as they did before — this is purely additional metadata, useful when a cycle's real-time span doesn't match its planned duration.
   - Internal: 5-second state heartbeat on every tick (AppLastAliveAt), and an AppDomain.ProcessExit safety net for ungraceful exits (crashes, Environment.Exit, OS shutdown). Worst case after an abrupt exit is now ~5 s of lost cycle progress, with the gap recorded as an AppShutdown pause on the cycle.
   - Internal: AppState's JSON serializer now emits enums as strings ("ScreenLock", "ManualPause", "AppShutdown") rather than integers, for readability and stability against enum reorderings.

v.1.1.0
   - New: configurable day-rollover time. Set when your day starts (default 06:00) in Settings; when the clock crosses the boundary — or the app resumes from a lock that straddled it — the cycle resets to your chosen start mode (Seated by default) and a "New day" toast appears.
   - All day-based stats now honor your rollover time instead of calendar midnight: today's ring, streak, heatmap, cycles-per-day chart, and the daily achievement counters all use it. A 02:30 standing cycle with a 06:00 rollover now counts toward the day you mentally finished, not the next one. The rollover itself never breaks your streak — even if it caught a cycle mid-flight.
   - New: "Always on top" toggle in Settings (default on). Also fixes the Dashboard window sometimes ending up behind the main disk on macOS.
   - First launch now drops straight into the Settings tab, with "Start with Windows" / "Start at login" pinned at the top tagged Recommended — replaces the old yes/no popup so configuration is visible, not buried.
   - New clock-face time picker for the rollover setting: two concentric rings (hours outer, minutes inner), drag / scroll / arrow keys to set, or click the central HH:MM and type the time directly.
   - Internal: streak / achievement / heatmap math is now parameterised on the rollover time, with TimeSpan.Zero as the calendar-day degenerate case. New CycleOutcome.RolloverReset distinguishes app-initiated resets from user resets.

v.1.0.7
   - Update checks now run on a regular timer rather than only when you start a sit/stand cycle, so a newly published release is picked up even while the app sits idle or paused.
   - Your sit/stand mode is now saved the moment you swap. An app restart — including an automatic update restart — resumes in the correct mode instead of occasionally repeating the previous standing cycle.
   - macOS now ships separate Apple Silicon and Intel installers (`osx-arm64` / `osx-x64`) instead of a single build that only worked on one architecture.
   - Clearer macOS install instructions: use System Settings → Privacy & Security → Open Anyway, and don't click "Move to Trash" on the first-open warning.

v.1.0.6
   - New: achievement count badge below the action toolbar on the main clock. Click it to jump to the Achievements tab.
   - The streak indicator above the clock is now clickable too — opens the Stats tab.
   - Pointer cursor on every clickable element of the main window, not just the clock.

v.1.0.3
   - Resilience: failed saves of state.json and history.jsonl no longer crash the app. A transient permission error or full disk is logged and ignored; the in-memory state remains authoritative and the next successful save catches up.
   - Dashboard window no longer leaks memory when opened and closed repeatedly — the dashboard ViewModel now properly unsubscribes from the update-checker on close.
   - macOS: the launch-at-login identifier is now dev.malforge.lazybones (was com.malforge.standup, a leftover from the project's original name). If you had Start at login enabled, the old LaunchAgent is cleaned up automatically the next time you toggle the setting.
   - Internal: a Lazybones.Tests project with 97 unit tests now ships alongside the app and runs in CI, gating releases. Covered units: TimeInputParser, StreakCalculator, HistoryStore, AchievementRules, AppState (including the StandUp→Lazybones data-directory migration), and the MVVM base classes.
   - Internal: substantial cleanup pass — dead code removed, error handling tightened to typed catches, lock consistency restored across HistoryStore reads, app-state persistence moved out of Features/Shell to Core/State to dissolve a 7-file import cycle, and macOS bundle id renamed for identity consistency.

v.1.0.2
   - Background update checks now run on every sit/stand cycle start (throttled to at most once per 30 minutes), so long-running sessions pick up new releases without restarting the app.
   - The daily goal is now a number of completed standing cycles rather than a number of standing minutes. The settings field reads "Daily standing cycles"; the outer ring and Today stat show cycle progress (with the minutes-stood-today shown as a secondary number). A cycle only counts if it ran to natural completion — bailing out via Swap doesn't count toward the goal, though the time you did spend standing is still tracked.
   - Streak follows the new cycle goal: a streak day is an active day that hit the cycle count. Days where the screen was locked all day or the app was paused are "paused" — neither hit nor miss. No mid-day instant-loss action; the streak fails at end-of-day if an active day finishes below the goal.
   - Old saved settings using the previous minutes-based goal are ignored; the new daily-cycle goal starts at its default. Lifetime achievements remain minutes-honest.
   - Stats tab: the 13-week minutes heatmap is now half-width, balanced by a new 14-day cycles-per-day bar chart on the right. The main-disk dashboard button switched from a cog to a stats glyph so the contents are less hidden.

v.1.0.0
   - First public release of Get Up, Lazybones!
   - Sit/stand reminder that lives as a round transparent desktop disk on Windows and macOS.
   - Layered progress rings track today's standing goal (outer) and the current cycle (inner).
   - Auto-pauses when you lock your screen and resumes on unlock — pause time is invisible to every metric.
   - Achievements covering streaks, daily intensity, response engagement, and lifetime volume.
   - Dashboard with stats, achievements, settings, and an updates tab.
   - Optional "Start at login" on both platforms (Windows Run key / macOS LaunchAgent).
   - Auto-update from GitHub Releases via Velopack — new versions show as a blue indicator on the main disk and a dedicated Updates tab.
